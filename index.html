弗尔帕斯塔原型CXXX-复读鸡

今天用半生不熟的C++做了个之前微博看到过的机器人，想要实现的目的便是“随机的情况下复读上一个人所说的话 并且大家还发现不了是机器人”，但是怎么说呢 做出来后 大家马上就能马上发现是机器人（不过也有大家早就事先知道骰子君是机器人的事情）。 经过一天的开发，目前实现的功能是2%的概率下随机复读以及当有两个人出现相同的发言，就会判定当前正在进行复读从而加入复读。 虽说是很简单的功能，但用自己完全不熟悉的C++实现起来完全不容易，光是指针就搞不懂到底是什么问题，中间出现好多次指针引发的错误… 同时也有一些逻辑问题纠结了好久，例如跟随复读功能，最开始如果别的群发了信息，那么判定中的“上一条信息”就会被覆盖，然后就判定失败，在尝试了许多办法之后，用了最笨的三个变量记录三条历史消息，这样如果别的群更新的信息在三个以内，仍然也可以复读，不过这个方法加的群多了就没用了，以后再改改好了。 不过现在还是成功做出来了，并且开源到了Github上，也成为了我的第一个开源项目，以后可以慢慢加些新功能进去，等完善的时候再把它放到人多的群，开始我的试验。 同时，复读鸡也是「弗尔帕斯塔原型计划」的第一款FP(Foerbasta Prototype)，解析世界的真理指日可待！Serwqa Kyono Lste. 于是乎，再拖了将近一个月，暑假的游戏开发计划终于开始了。这次想要做的是两年前暑假曾经做过的Roguelike CRPG游戏LAISDIA~Million Feat Adventures的复刻版（其实之前的源码丢了所以重新写一遍），不过这次的名字更简单了：LAISDIA~Exploeres 这次一次开发，有了之前的经验，无论是代码结构还是简洁性，都有了很大的提升，而且也只用了半天时间就完成了以前三天所需要实现的功能（大概） 为了避免之前LAMFA最后的悲剧：代码混乱，毫无可读性，这一次重写我注重了每个功能写成单独类的模块化，并且大量使用注释——良好的习惯是成功的先决条件。 今天进度：

Twenty Seventeen

2017这个和自己挺喜欢的年份数字就这样过去了，总归还是要写点什么的。而且不能像去年那样，只是个游戏的年度总结，2017发生这么多事，总得多写点东西。 那么伴随着新入坑的ｖａｐｏｒｗａｖｅ音乐，写些什么呢？

ＳＣＨＯＯＬＷＯＲＫ＼

00:00

仿佛是在开玩笑一般，匆匆忙忙从决定想学画画到坐到画室中的间隔连1周都不到。至此，世界线应该是在当初选择学文科之后的第二次大变动吧，现在为了实现梦想可以说是“曲线救国”了。 学画画总归还是有点用的，起码现在自己想画点什么的时候也有能力去画了（然而还远远不够）。 不过虽然现在统考成绩还没出来，但自己心里也有点B数了…(如果能在给我一年….) 怎么说呢学业这一块，感觉还是一如既往吧（苦笑）。

ＤＥＶＥＬＯＰＭＥＮＴ＼＼

今年真的大概是一行代码都没写过了，无论是之前忙活了两年的Darkarena还是新作品，一　行　都　没　有　写　。 原因大概是学画画和DOTA2，夺走了我大部分时间。 2016是ZINC忙碌的一年，2017则是ZINC碌碌无为的一年。

挖坑历史_

2011-2012 TASTAR Games:冒险星Venture Star 宝石战记 2013 Powermessagin:莱斯迪亚传说 2014 Ofrinca:莱斯迪亚传说New Adventure Nexunity:Darkarena alpha1 2015 Nexunity:Darkarena alpha2 Ch’s:Darkarena alpha3 2016 ZINC:Darkarena alpha4 LAISDIA~Million Feat Adventures MagicMiracleDuel 2017 NOTHING 那么问题来了：以上列表里 我的工作室换了几次名字？ 真正能玩的游戏有几个？ 不过18年了，当初小学的计划还是今年推出LAISDIA正传，在游戏圈惊奇波澜的…. 今年还是开个新坑吧：LAISDIA~Explorers 没错 就是它，终于开始做了（苦笑）。 怀念一下弃坑的Darkarena，最接近完成的作品。

ＧＡＭＥＰＬＡＹ＼＼＼

今年买了好多游戏，还是不要再一一点评了，不过今年游戏方面的重点也不是单机游戏了，而是DOTA2。 DOTA2虽然早在13年就建了号，但是认真玩起来应该是从16年底开始。一向很讨厌MOBA类游戏的我，在DOTA2中一年的消费直逼玩了三年的FF14，为什么会深陷其中呢？我自己也搞不太清楚，这大概就是DOTA2的魔力吧（G胖的魔力）。不过其中应该也有个原因：满足我“求异”的心理。 我自己的爱好总是小众的，并且去刻意追求“跟大家不一样”。例如别人都玩英雄联盟，我玩DOTA2。别人都用Android ios，我用Windows Phone（虽然现在也不用了），别人都把Taskbar放在屏幕底下，而我放在顶部，别人宝可梦队伍都用洛杉矶、沙暴队等强力队伍，而我偏爱受队和雪天队等等等等。 因为这点，自己吃了不少亏，不过自己也愿意这样了。昨天又发现了新的爱好：Ｖａｐｏｒｗａｖｅ。 停下，有点跑题了。 今年买的但根本没怎么玩的游戏（喜+1）：Bastard Bonds（给游） The Black Watchmen（装逼） Cities:Skylines（天国的模拟城市） Life is Strange（补票） Planet Coaster（天国的过山车大亨（和大雨）） Stardew Valley（养老） 精灵宝可梦：究极之日（以及同样是背着爸妈买的3DS） 还有感谢最好的朋友土豆送的Armello。

ＳＯＵＴＨＰＡＲＫ＼＼＼＼

今年还看了久仰大名的南方公园，SP真的是对我影响很大了，所以单独开一个栏目。 感觉自己的性格在看过SP之后彻底发生了改变，如果说以前自己的“本我”和“超我”能够打平的话，现在已经是“本我”占上风了。上图的Cartman的”Whatever! I’ll do what i want.”也成为了我的的一部分。 准确来说，这一切的发生都是在：看SP、结识捏他、周阴婷和淋淋的共同影响下发生的。 虽然不想接受，但自己好像越来越像自己讨厌的Cartman了。 希望“超我”在2018还能时不时出来捣一下乱，不要让“Cartman本我”太过分了。

Our Fate ~ Life is Strange 二周目有感

Don’t you ever forget me, Maxine Caufield.

这是第二次玩LIS了，主要是为了重新回顾一下这么优秀的剧情沉浸体验，二是为了在风暴前夕最终章之前能回顾一下本篇的剧情，三是为了发掘一下之前玩的时候没有发现的细节。 这次仍然选择了牺牲Chloe的结局，并不是我讨厌Chloe。虽然我并没有选择能够促进Chloe和Max的爱情的选项，因为我认为Chloe应该和Rachel在一起，Max则是Waren。

The Golden Hour

而选择这个结局的真正原因是我在《蝴蝶效应》和《魔法少女小圆》（《命运石之门》不算，因为我觉得在其中并没有体现出“人无法与既定的命运抗争”）影响下形成的时空观，即：“宇宙、命运、人的各个选择都是在宇宙诞生的那一刻早已注定好的，并且互相联系。一旦改变，势必将引发”蝴蝶效应“，导致整个宇宙的改变。对过去的干涉越多，那么后果就越严重，就例如游戏内的Alisa，起初只是被球砸到，到最后遇到的麻烦越来越大。死亡终究是Chloe的结局，正如Chloe在最后所说：”This is my fate, our fate.” 而这个结局的CG也是最让我难忘的，不像另一个结局的CG，这个CG是最让人感动的。伴随着那一声不可挽回的枪响，Max的身躯随之一震，BGM Spanish Sahara的插入，随后Max与Chloe相处的照片的消逝，是最大的泪点。 说起泪点，其他两次让我哭出来的分别是选择结束平行次元Chloe生命的时候和之后目睹Chloe的父亲走出家门的时候的Chloe的那句”She’s never leaving me.”和William的” That make all of us.”，最终两人都离开了Chloe。 而另一次，就是发现Rachel尸体的那段了。

Rachel was(is) here.

Chloe的”I love her so much! How can she be dead!”伴随着BGM Moutains。Chloe与Max的调查最终带来的绝望的真相让人无法接受。在玩风暴前夕之后，我更无法接受Rachel这样好的女孩死去的事实，Chloe继Max之后，又失去了Rachel，那样的痛苦，我可以感受得到。 最让我难忘的片断其实还有一个，就是第五章最后Max在暴风雨中驾车前往双鲸餐厅，听Nathan之前的电话录音的那段。BGM The Sense Of Me中的”Try to try again”正反映着Max一遍一遍去拯救Chloe却又一遍一遍失败，也正是如此奠定了我选择的结局——Chloe的命运无法被改变。 同样，Nathan真的是个可恨又可怜的悲剧角色，第一次玩的时候，在这一段真的很同情一直被利用却又被另类对待的Nathan。 第二遍玩因为剧情缘故，体验虽然不如一周目，但是仍然让我很享受这一回顾的过程。LIS和口袋妖怪不可思议的迷宫以及其他那些优秀的游戏一样，让我每隔一段时间就会回顾。这些游戏都有共同的特点：即便没有很高的重复游玩价值，但是凭借着优秀的剧情和配乐就如此吸引人。LIS在这两点更为突出。 即便被认为是烂尾，LIS仍然因为出色的配乐、令人难忘的剧情、沉浸的电影式体验成为我心目中最出色的剧情互动式游戏。 现在，期待风暴之前的第三章和真正的续作，不知道能不能给我们带来新的结局。

记录~Visual Editor与Parsoid的安装

记录一下自己安装Mediawiki插件Visual Editor以及Parsoid的过程。

安装Node.js

Parsoid需要node版本大于4.x，因为Debian下默认node版本太低，需要更高的版本，索性安装最新的8.x。

curl -sL https://deb.nodesource.com/setup_8.x | bash -
apt-get install -y nodejs

通过node -v来确认版本

安装Parsoid

首先添加Mediawiki的源

apt-key advanced –keyserver keys.gnupg.net –recv-keys 90E9F83F22250DD7
echo “deb https://releases.wikimedia.org/debian jessie-mediawiki main” | tee /etc/apt/sources.list.d/parsoid.list

然后安装Parsoid，由于上一步添加的源是https地址，所以需要安装apt-transport-https包来让apt支持https。

apt-get install apt-transport-https
apt-get update && apt-get install parsoid

配置Parsoid

Parsoid配置文件位于/etc/mediawiki/parsoid/config.yaml，需要进行编辑。 修改第35行：

uri: ‘http://wiki域名/api.php'

Parsoid的默认端口是8142。 随后重启Parsoid:

service parsoid restart

安装Visual Editor

进入Mediawiki的extensions目录,使用git同步。

git clone https://gerrit.wikimedia.org/r/p/mediawiki/extensions/VisualEditor.git
cd VisualEditor
git submodule update –init

修改LocalSettings.php,加入以下代码启用Visual Editor：

$wgMFAutodetectMobileView = true;

// Enable by default for everybody
$wgDefaultUserOptions[‘visualeditor-enable’] = 1;

// Optional: Set VisualEditor as the default for anonymous users
// otherwise they will have to switch to VE
// $wgDefaultUserOptions[‘visualeditor-editor’] = “visualeditor”;

// Don’t allow users to disable it
$wgHiddenPrefs[] = ‘visualeditor-enable’;

// OPTIONAL: Enable VisualEditor’s experimental code features

#$wgDefaultUserOptions[‘visualeditor-enable-experimental’] = 1;
$wgVirtualRestConfig[‘modules’][‘parsoid’] = array(
// URL to the Parsoid instance
// Use port 8142 if you use the Debian package
‘url’ => ‘http://localhost:8142',
// Parsoid “domain”, see below (optional)
‘domain’ => ‘wiki域名’,
// Parsoid “prefix”, see below (optional)
‘prefix’ => ‘localhost’
);

至此Parsoid于Visual Editor安装成功并启用。

临摹～ 风景画

这个星期，为了写生画风景画而准备，画室每天都在让我们临摹大师的风景画。比起色彩静物，我觉得我反而更喜欢画风景画（不过大家都这么感觉）。因为画风景画不用刻意去画笔触，还可以按照自己的理解去画，即使出现了错误的颜色或者笔触，有些也不会影响画面整体的感觉，到最后整幅画面看起来还是很棒。 第一张画的感觉不错的画，并且同时也发现了自己擅长的事物——花与树，我发现自己擅长去寻找那些树叶或者花朵的颜色和笔触。 画这张画的时候其实很有趣，在点树的时候，因为太困了就陷入了半醒半睡的状态，当某一刻自己突然清醒的时候，发现整棵树的感觉已经出来了，非常神奇。 第二张，在尝试了16k的树之后，开始尝试更大一点的8k尺寸，并且画了花。在这幅画中，我彻底的形成了我画植物的风格（其实就是瞎点）。其实我觉得并没有第一张画好看，但是大家都说这张更好看一点。 第三张，开始尝试风景，但是这张画不是很成功，暗色的云的颜色的难度超乎我的想象，并且这张画老师也改过。 最后一张，个人感觉一般，树林和草地刻画不是很细致，但是效果貌似也出来了。另外这张的云感觉很棒。 其实这几张画的原作者是谁都不知道，但是经过我的临摹之后，画也被我改动了一番来迎合我的风格（其实就是临的不像）。这下以来，我算是找到了自己喜欢的东西了，所以很期待明天的写生，并且写生可以算是“创作”了，希望也能画出不错的效果。

Electricity

今天设计课学了抽象，于是诞生了下面这幅画作： 这幅画我相信和其他抽象派作品一样，如果不予以解读，那么大家应该都看不懂。 之所以命名为《Electricity》是有原因的，一是最近一直在听的歌曲《Locus》中的歌词：

Break free! Two-phase, two-phase electricity.

并且，画面最开始是由下面的元素构成的： 我们可以看到 左下角是一台Mac电脑（可以从上方状态栏和下方的Dock栏看出），左上角是一台路由器（旋转90°），右上角则是一部手机。画面整体由大大小小的矩形构成，代表着电子产品。同时，在左下角和右上角繁乱的曲线则代表电子产品内复杂的内容与电流，并且曲线连接了各个矩形，代表着信息在不同设备之间的交流。 能扯这么多 我也是厉害。
